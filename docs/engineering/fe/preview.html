Notification System

Real-time notification bell with popover dropdown, notification list page, error/empty states, and design-system-compliant spinners. Refactored to eliminate duplicate socket listeners.

Architecture (What Was Built)
Backend: NotificationService → channels (IN_APP, EMAIL, SMS, PUSH) in parallel. Fire-and-forget from booking/trip/admin services.
Frontend: SocketConnector (global, in Providers) → Zustand store → React Query hooks → NotificationBell (Popover) + /notifications page.
Socket: Single connection per session. SocketConnector connects on auth, disconnects on logout. No duplicate listeners.
Polling: Unread count polled every 30s as fallback. Real-time via notification:new socket event.

Desktop header: single bell icon with unread badge. Click opens a Popover (not raw dropdown) with recent notifications. Mobile: simple link to /notifications page.
